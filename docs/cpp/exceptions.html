Exceptions¶

The CVC5 API communicates certain errors using exceptions. We broadly distinguish two types of exceptions: CVC4ApiException and CVC4ApiRecoverableException (which is derived from CVC4ApiException).

If any method fails with a CVC4ApiRecoverableException, the solver behaves as if the failing method was not called. The solver can still be used safely.

If, however, a method fails with a CVC4ApiException, the associated object may be in an unsafe state and it should no longer be used.
